# Contributing

Thank you for your interest in contributing to this project.

This repository provides a command-line interface built on top of
reference-grade cryptographic primitives. Contributions are expected
to prioritize correctness, clarity, and reproducibility.

## Ground rules

- Follow the Code of Conduct.
- Keep changes minimal and well-scoped.
- Prefer explicit behavior over implicit defaults.
- Avoid introducing unnecessary dependencies or side effects.

## What to submit

### Bug reports

Please include:
- Exact command invocation
- Input parameters and flags
- Expected vs actual behavior
- Relevant output or error messages
- Environment details (OS, architecture, version, commit)

### Feature requests

Please describe:
- The use case or limitation being addressed
- Proposed CLI interface changes
- Backward compatibility considerations
- Security or misuse implications

### Security issues

Do **not** report suspected vulnerabilities via public issues.

Use the private security advisory mechanism as described in `SECURITY.md`.

## Development workflow

1. Fork the repository
2. Create a scoped branch (`feature/<name>` or `fix/<name>`)
3. Keep commits focused and descriptive
4. Open a pull request with clear context and rationale

## Verification

Contributors are responsible for verifying their changes.

If automated tests are not available, include a concise description
of manual verification performed.
